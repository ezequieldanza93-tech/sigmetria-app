# Testing End-to-End (Playwright)

## Objetivo
Infraestructura de testing E2E con Playwright para cubrir flujos críticos: autenticación, CRUD de empresas, navegación de establecimientos, y formularios clave. Integrado con CI y con estrategia de base de datos aislada.

## Stack
- Playwright (última versión estable)
- Supabase (auth + DB via service role para test setup)
- GitHub Actions (CI existente `.github/workflows/ci.yml`)

## Decisiones tomadas

| Decisión | Opción |
|----------|--------|
| Framework | **Playwright** (vs Cypress). Cross-browser nativo, más rápido en CI, mejor DX con Next.js, parallel execution nativo |
| DB para tests | **Branch de Supabase** — crear branch `e2e-tests` desde `main`, correr migrations, seed data de prueba |
| Auth en tests | Login con `supabase.auth.signInWithPassword()` vía API — test user pre-creado en seed |
| Page Objects | Sí — patrón Page Object Model para flujos reutilizables |
| CI | Job separado `e2e` en CI, con preview deployment o contra la branch de Supabase |
| Reportes | Playwright HTML Report + GitHub Actions Report |
| Test data | Seed script con datos de prueba (consultora, empresas, establecimientos, usuarios) |

## Page Objects

### `e2e/pages/login.page.ts`

```typescript
export class LoginPage {
  constructor(private page: Page) {}

  async goto() { await this.page.goto('/login') }
  async fillEmail(email: string) { /* ... */ }
  async fillPassword(password: string) { /* ... */ }
  async submit() { /* ... */ }
  async loginAs(email: string, password: string) {
    await this.goto()
    await this.fillEmail(email)
    await this.fillPassword(password)
    await this.submit()
    await this.page.waitForURL('/dashboard')
  }
}
```

## Auth Setup — Storage State

Usar `globalSetup` o `setup project` para login una vez y reusar storageState en todos los tests:

```typescript
// e2e/setup/auth.setup.ts
import { test as setup } from '@playwright/test'
import { LoginPage } from '../pages/login.page'

const authFile = 'e2e/.auth/user.json'

setup('authenticate', async ({ page }) => {
  const login = new LoginPage(page)
  await login.loginAs(process.env.E2E_TEST_USER_EMAIL!, process.env.E2E_TEST_USER_PASSWORD!)
  await page.context().storageState({ path: authFile })
})
```

En el `playwright.config.ts`, agregar `storageState: 'e2e/.auth/user.json'` como dependencia.

## Especificaciones de Tests — Scenarios

### Auth (`e2e/specs/auth.spec.ts`)

| Test | Descripción |
|------|-------------|
| Login exitoso | Email + password → redirect a /dashboard |
| Login fallido | Credenciales inválidas → mensaje de error |
| Protección de ruta | Ir a /dashboard sin auth → redirect a /login |
| Logout | Click logout → redirect a /login |

### Empresas (`e2e/specs/empresas/`)

| Test | Descripción |
|------|-------------|
| Listar empresas | Ver listado con datos del seed |
| Crear empresa | Completar formulario → ver en listado |
| Ver detalle empresa | Click en empresa → ver tabs |
| Editar empresa | Cambiar razón social → ver cambios |
| Eliminar empresa | Eliminar → ya no aparece en listado |

### Establecimientos (`e2e/specs/establecimientos/`)

| Test | Descripción |
|------|-------------|
| Navegar tabs | Click en cada tab → contenido visible |
| Ver datos del seed | Los datos cargados en seed se muestran correctamente |

## CI Integration

Modificar `.github/workflows/ci.yml`:

```yaml
jobs:
  # ... existing quality job ...

  e2e:
    needs: quality
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm
      - run: npm ci
      - name: Install Playwright browsers
        run: npx playwright install --with-deps chromium
      - name: Seed E2E test data
        run: npm run seed:e2e
        env:
          SUPABASE_URL: ${{ vars.E2E_SUPABASE_URL }}
          SUPABASE_SERVICE_ROLE_KEY: ${{ secrets.E2E_SUPABASE_SERVICE_ROLE_KEY }}
      - name: Run Playwright tests
        run: npm run test:e2e
        env:
          E2E_BASE_URL: http://localhost:3000
          E2E_TEST_USER_EMAIL: test@sigmetria.e2e
          E2E_TEST_USER_PASSWORD: ${{ secrets.E2E_TEST_USER_PASSWORD }}
          NEXT_PUBLIC_SUPABASE_URL: ${{ vars.E2E_SUPABASE_URL }}
          NEXT_PUBLIC_SUPABASE_ANON_KEY: ${{ vars.E2E_SUPABASE_ANON_KEY }}
      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: playwright-report
          path: playwright-report/
          retention-days: 7
```
